# 项目规格说明

## 1. 项目概述
TutoringSys 是一个基于大语言模型的智能作业批改与辅导系统，采用前后端分离的 B/S 架构。系统旨在通过自动化技术辅助教师完成作业批改，并为学生提供即时、详细的批改反馈。核心功能包括：客观题（单选、多选、判断）的自动批改、主观题（短文本、富文本）的大模型智能批改、结构化批改报告呈现、教师学情可视化分析、多角色权限管理等。

## 2. 系统架构
- **后端**：Spring Boot 3.0 + Java 17，提供 RESTful API，集成 Spring Security 实现认证授权，MyBatis-Plus 作为持久层框架，与 MySQL 8.0 交互。
- **前端**：Vue 3 + TypeScript，使用 Pinia 状态管理，Element Plus 组件库，ECharts 实现数据可视化。
- **数据库**：MySQL 8.0 存储用户、课程、作业、题目、批改记录等核心业务数据。
- **缓存**：Redis 7.0 用于管理登录 Token、缓存高频错误分析结果、热门辅导方案，提升系统响应速度。
- **大模型集成**：调用阿里 Qwen2.5-Max API 进行主观题智能批改，通过提示词工程引导模型生成结构化反馈。

## 3. 功能需求

### 3.1 学生端
- **作业列表**：查看已选课程的所有作业，按状态（待提交、已批改）筛选。
- **作业提交**：
  - 客观题：通过表单提交单选、多选、判断题答案。
  - 主观题：使用轻量级富文本编辑器（如支持简单格式）提交答案。
- **查看批改报告**：提交后生成结构化报告。
  - 客观题：标注对错，显示标准答案。
  - 主观题：展示大模型生成的批改评价、核心错误点、简要修正建议。
- **个人中心**：修改头像、用户名、密码；查看已绑定课程；查阅历史作业提交与批改记录。

### 3.2 教师端
- **课程管理**：接收管理员分配的课程，管理课程内的学生名单（增删）。
- **作业发布**：创建多类型作业，设置题目、分值、截止时间，并可发布通知。
- **批改复核**：批量查看智能批改结果，支持人工复核与评分调整，补充评语。
- **学情可视化**：通过 ECharts 图表查看班级作业错误分布、知识点掌握热力图，筛选高频错误题型。
- **数据导出**：导出批改明细（Excel/CSV）。

### 3.3 管理员端
- **用户管理**：增删改查学生、教师账号，分配角色权限。
- **课程管理**：增删改查课程，将课程分配给教师，维护课程与学生的关联。
- **题库管理**：管理客观题题目库，配置批改规则（如分值、答案）。
- **系统监控**：查看历史作业提交及批改记录，监控系统性能指标（如 API 调用次数、响应时间）。

## 4. 技术实现要点

### 4.1 多类型作业智能批改流程设计
- **客观题**：前端提交答案后，后端直接与标准答案比对，判断对错，并返回正确结果。
- **主观题**：后端调用大模型 API，传入学生答案和题目要求，通过提示词工程引导模型分析答案的逻辑完整性、核心要点，识别错误并生成修正建议。结果解析后存入数据库。
- **拓展性**：后续可考虑支持编程题批改（作为可选功能）。

### 4.2 大模型集成与提示工程优化
- **API 集成**：后端封装统一的 LLM 服务模块，处理与阿里 Qwen2.5-Max 的通信，包含错误重试、超时控制、安全过滤。
- **提示工程**：针对不同题型（如简答、论述）设计专门的提示模板，包含评分标准、示例答案、输出格式（JSON），引导模型生成结构化反馈。  ```
- **缓存优化**：对相同题目和相似答案的批改结果进行缓存（Redis），避免重复调用大模型，降低开销。

### 4.3 前后端分离的系统工程实现
- **后端**：
  - 基于 Spring Boot 构建业务模块：用户认证（JWT）、作业管理、批改服务、大模型调度。
  - 使用 Spring Security 实现基于角色的权限控制（学生/教师/管理员）。
  - 通过 Redis 存储登录 Token 和缓存数据。
- **前端**：
  - Vue 3 + TypeScript 构建 SPA 应用，采用组合式 API。
  - Pinia 管理全局状态（用户信息、课程列表等）。
  - Element Plus 实现 UI 组件，适配学生、教师、管理员三种操作界面。
  - 富文本编辑器采用轻量级库（如 TinyMCE 或 wangEditor）。

### 4.4 数据可视化与学情分析
- **学生端**：个人批改报告以简洁的卡片形式展示，客观题用对勾/叉号标识，主观题用文字列表展示错误点和建议。
- **教师端**：使用 ECharts 绘制班级错题分布柱状图、知识点掌握热力图，支持按题目、学生筛选，快速定位班级薄弱环节。
- **管理员端**：系统概览仪表盘，展示用户总数、课程数、作业提交量、批改调用次数等，辅助全局管理。

### 4.5 数据缓存加速
- **Redis 应用**：
  - 登录 Token：设置过期时间，减少数据库查询。
  - 高频错误分析结果：对同一课程同一题目的错误统计结果缓存一段时间。
  - 热门辅导方案：对常见错误预先生成辅导建议缓存。
- 作业提交和报告查看操作响应时间控制在 1 秒以内。

## 5. 技术指标
1. 系统支持两种核心作业类型（客观题/主观题）的在线填写、提交与智能批改。
2. 客观题批改能判断对错并给出正确答案；主观题批改能识别核心错误点并提供修正建议。
3. 前后端分离架构，API 设计遵循 RESTful 规范。
4. 系统并发支持至少 100 用户同时访问，批改请求平均响应时间 ≤3 秒（含大模型调用）。
5. 大模型 API 调用失败时提供降级处理（如返回提示“批改服务繁忙，请稍后重试”）。
